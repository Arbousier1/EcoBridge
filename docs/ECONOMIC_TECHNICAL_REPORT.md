## 1. 这东西是干嘛的

Minecraft 服务器的经济系统有个永恒的问题：

> 玩家不停地挖矿、种田、打怪 → 物品越来越多 → 物价越来越低 → 钱不值钱 → 经济崩了

EcoBridge 做的事情很简单：

```
物品太多 → 自动降价（鼓励玩家少生产）
物品太少 → 自动涨价（鼓励玩家多生产）
价格太低 → 系统托底收购（防止崩盘）
```

就像现实中的中央银行调控经济一样，只不过这是给 Minecraft 用的。

## 2. 经济是怎么运转的

想象一个水池：

```
         ╔══════════════════╗
   水龙头║    FAUCET        ║  水龙头 = 产生钱的途径
   进水 →║  (金币注入)      ║  玩家卖东西给系统商店 = 钱进了服务器
         ╠══════════════════╣
         ║                  ║
         ║    M1 货币池     ║  水池 = 服务器里流通的总金币
         ║    (钱的总量)     ║
         ║                  ║
         ╠══════════════════╣
   排水口║    SINK          ║  排水口 = 消耗钱的途径
   出水 ←║  (金币回收)      ║  税收、系统商店出售物品 = 钱从服务器消失
         ╚══════════════════╝
```

**目标**：保持水池的水位（金币总量）稳定在 1000 万左右。水太多 → 通货膨胀。水太少 → 通货紧缩。

EcoBridge 每天自动调节水龙头和排水口，让水位保持平稳。

## 3. 价格是怎么算出来的

每个物品的价格由一个公式决定：

```
价格 = 基础价 × 环境因子 × e^(-弹性 × 有效供应量)
```

别被公式吓到。翻译成人话：

| 参数 | 什么意思 | 举个例子 |
|------|---------|---------|
| **基础价** | 你在配置里设的初始价格 | 钻石 = 100 金币 |
| **环境因子** | 季节性波动、周末效应、新手优惠 | 周末人多，价格涨 5% |
| **弹性** | 价格对供应量的敏感程度 | 钻石稀缺所以弹性低（价格稳定），圆木泛滥所以弹性高（价格敏感） |
| **有效供应量** | 过去 N 天里该物品的交易量加权 | 最近 7 天内卖出了 1000 个 = 供应量大 |

**举例**：
- 基础价 100 金币的钻石，供应量 500 个 → 价格约 95 金币（微跌）
- 基础价 1.5 金币的圆木，供应量 50000 个 → 价格约 0.6 金币（大跌）

## 4. 价格跌了怎么办——三层保护机制

### 第一层：均值回归恢复

```
当价格 < 历史均价的 85% 时
  → 系统自动向上拉价格
  → 跌得越久，拉力越大（积分效应）
  → 直到回到历史均价的 92%
```

就像弹簧：价格被压得越低，回弹力越强。

### 第二层：波动率感知底价

```
底价 = 历史均价 × 62% × 波动率倍率
```

- 市场平静时：底价偏低，让市场自由调节
- 市场波动时：底价自动抬高，防止恐慌性抛售

### 第三层：系统保底收购（v2.0 新增）

```
系统收购价 = max(基础价 × 40%, 历史均价 × 20%, 0.01)
```

**不管发生什么，每个物品都有一个绝对不会跌破的价格。** 这就像现实中的「央行托底」——OSRS（RuneScape）的 High Alchemy 魔法也是这个原理。

```
例子：钻石基础价 100 金币
  → 系统收购价 = 100 × 40% = 40 金币
  → 无论市场怎样，服务器永远以 40 金币收购钻石
  → 钻石价格永远不会低于 40 金币
```

## 5. AI 大脑——可选的智能调控

除了纯数学算法，EcoBridge 还有一个可选的 AI 协同模式：

```
纯算法模式（默认）：
  经济数据 → 模糊逻辑 + PID + MPC → 调控决策

AI 协同模式（可选）：
  经济数据 → 算法先给安全方案 → AI 给建议 → 混合执行
  AI 挂了？自动切回纯算法，不中断服务
```

支持 Claude API 和 OpenAI API。AI 权重可在 0%（纯算法）到 100%（纯 AI）之间调节。

**推荐设置**：权重 30%（30% AI 建议 + 70% 算法决策），既安全又智能。

## 6. 技术架构——怎么做到的

```
┌─────────────────────────────────┐
│        Minecraft 服务器          │
│  ┌───────────────────────────┐  │
│  │   UltimateShop (商店插件)  │  │
│  └───────────┬───────────────┘  │
│              │ 价格拦截          │
│  ┌───────────▼───────────────┐  │
│  │     EcoBridge Java 层      │  │
│  │  - 事件监听                │  │
│  │  - 交易审计                │  │
│  │  - AI 协同                 │  │
│  │  - 配置管理                │  │
│  └───────────┬───────────────┘  │
│              │ Panama FFM 调用   │
│  ┌───────────▼───────────────┐  │
│  │   EcoBridge Rust 核心      │  │
│  │  - 定价引擎 (微秒级)        │  │
│  │  - GARCH 波动率建模         │  │
│  │  - 卡尔曼滤波               │  │
│  │  - MPC 预测控制             │  │
│  │  - 交易风控（傀儡检测）      │  │
│  │  - SIMD 加速计算（AVX2）    │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

**为什么要用 Rust？** 价格计算要在玩家打开商店菜单的瞬间完成，必须极快。Rust 的 SIMD（单指令多数据）可以在一次 CPU 指令里同时算 4 个物品的价格，比纯 Java 快 3-5 倍。

**数据存哪里？** 
- 交易记录 → MariaDB（持久化）
- 实时价格缓存 → H2 嵌入式数据库（内存级速度）
- 跨服同步 → Redis（可选）
- 内存热层 → Rust 内存（微秒级访问）

## 7. 配置指南

你只需要关心这几个参数：

```yaml
economy:
  # 时间衰减天数：越小价格恢复越快
  # 推荐：稀有物品 3-5 天，常见物品 2-3 天
  tau: 7.0

  # 价格弹性：越大价格对供应越敏感
  # 推荐：钻石 0.003，圆木 0.02
  default-lambda: 0.002

  # 卖出占比：玩家卖东西的价格 = 买入价 × 这个比例
  sell-ratio: 0.58

  # 均值回归恢复
  recovery:
    enabled: true
    activation-ratio-to-history: 0.85  # 跌到 85% 开始拉
    target-ratio-to-history: 0.92      # 拉到 92%
    strength: 0.34                     # 拉力强度

# AI 协同（不配 API Key 就不启用）
ai-co-pilot:
  enabled: false
  provider: "claude"    # claude 或 openai
  api-key: ""           # 从环境变量 ANTHROPIC_API_KEY 读取也行
  weight: 0.30

# 数据库密码（优先读环境变量，安全！）
# 环境变量：ECOBRIDGE_DB_PASSWORD
database:
  password: "change_me_to_real_password"
```

## 8. 给服主的建议

### 新手服主（< 50 人在线）

直接用默认配置就行。EcoBridge 的默认值已经为小型服务器优化过了。

### 中型服主（50-200 人在线）

建议调整：
- `tau: 5.0` — 供应衰减加快，价格更敏感
- `sell-ratio: 0.55` — 卖出价稍低，控制金币产出
- `default-lambda: 0.003` — 略微提高弹性

### 大型服主（> 200 人在线）

建议：
- 启用量子限额（`player-market.quota.enabled: true`）— 限制每个玩家每天的卖出量
- 启用 Redis 跨服同步 — 多子服共享经济数据
- 考虑启用 AI 协同模式（`ai-co-pilot.enabled: true`）— AI 可以在复杂情况下给出更好的决策

### 常见问题

**Q: 为什么某个物品价格一直在跌？**
A: 卖出量 > 买入量。要么减少供应（限制玩家每天卖出量），要么提高 tau（加快衰减），要么系统底价会兜底（不低于 40%）。

**Q: 如何让某个物品价格更稳定？**
A: 在 `items.yml` 里单独设置该物品的 `lambda` 值。lambda 越小，价格越稳定。

**Q: 玩家抱怨价格太低怎么办？**
A: 提高 `sell-ratio`（比如从 0.58 提到 0.65），玩家卖出价会更高。

| 指标 | 含义 | 健康范围 |
|------|------|---------|
| M1 供应比率 | 实际金币总量 / 目标总量 | 0.80 - 1.20 |
| 价格指数 | 所有物品加权平均价格 / 基准 | 0.70 - 1.30 |
| 通胀率 | 价格指数的年化变化率 | -2% ~ +5% |
| 市场热度 | 交易活跃度指标 | 2.0 - 8.0 |
| 波动率 (GARCH) | 价格波动程度 | < 0.10 = 稳定 |
